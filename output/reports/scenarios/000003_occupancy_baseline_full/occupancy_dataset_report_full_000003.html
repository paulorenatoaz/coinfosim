CoInfoSim — Occupancy Dataset Report

Purpose: Dataset provenance, preprocessing protocol, and reproducibility record for the Occupancy Detection scenario.

Scope. This report documents the raw data source, local file roles, SHA-256 hashes, preprocessing steps, and leakage controls used in the CoInfoSim Occupancy Detection scenario. Monte Carlo simulation results are in the arm-specific reports.

2. Citation and external source

L. Candanedo and V. Feldheim.
"Accurate occupancy detection of an office room from light, temperature, humidity and CO₂ measurements using statistical learning models."
Energy and Buildings, Volume 112, 2016.

Ground-truth occupancy was obtained from timestamped photographs taken every minute. The dataset supports binary room-occupancy classification using temperature, humidity, light and CO₂ sensors.

11. Leakage-control notes

  • Train-only standardization: channel means and standard deviations are estimated exclusively from datatraining.txt. The same parameters are applied to both the training pool and the fixed test split.
  • Fixed test split: datatest.txt and datatest2.txt are concatenated into a single fixed evaluation split that is never used for fitting, standardization, or model selection.
  • No test information in Gaussian/GMM fitting: the single-Gaussian and GMM class-conditional models are fitted exclusively from the standardized training pool. No test rows participate in parameter estimation.
  • Balanced training sampling: Monte Carlo replications draw balanced samples from the training pool using a deterministic seed, independently of the test split.
