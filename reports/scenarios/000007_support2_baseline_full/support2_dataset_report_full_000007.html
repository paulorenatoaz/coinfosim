CoInfoSim — SUPPORT2 180-Day Mortality Dataset Report

Prediction target: death within 180 days after SUPPORT2 study entry. This implementation follows the original SUPPORT six-month prognostic objective.

1. Dataset overview and provenance

SUPPORT2 contains 9,105 seriously ill hospitalized adults from five United States medical centers. The canonical source is the Vanderbilt University Department of Biostatistics repository, mirrored by the UCI Machine Learning Repository. DOI: 10.3886/ICPSR02957.v2.

2. Raw structural anomaly and reconstructed patient ID

The source header advertises 47 names while every data row has 48 fields. The unnamed leading field is reconstructed explicitly as the integral, unique patient id running from 1 through 9,105; no implicit pandas index behavior is used.

3. Original prognostic objective and fixed endpoint

The original SUPPORT work sought prognostic estimates over a six-month (180-day) period. CoInfoSim therefore uses a fixed, interpretable horizon instead of eventual death:

death_180d = ((death == 1) & (d.time <= 180)).astype(int)

Day 180 is inclusive: a recorded death on day 180 is positive; a death after day 180 is negative for this endpoint. death_180d is derived after schema validation and before complete-case filtering or splitting; it is not a raw CSV column.

death is the eventual death indicator and d.time is follow-up/event time. All 2,904 patients with death == 0 have defined follow-up beyond the horizon; the minimum survivor d.time is 344 days and zero survivors have d.time <= 180. This survivor-follow-up validation is required before assigning them class 0.

5. Why hospdead is not the target

hospdead is not used as the primary target or as a predictor. It describes in-hospital death, which is a different estimand from the approved fixed 180-day outcome and does not implement the original six-month prognostic objective.

7. Excluded variables and leakage controls

death, d.time, death_180d, hospdead, surv2m, surv6m, id, and dzgroup never enter predictor data. dzgroup is used only for stratification and reporting. No target-source, alternative-outcome, post-baseline survival estimate, identifier, or disease-group field enters standardization, Gaussian/GMM fitting, subset enumeration, or classifiers.

8. Raw target counts and survivor audit

Raw rows: 9,105. Raw classes: 4,840 negative and 4,265 positive. Survivor count: 2,904; minimum survivor follow-up: 344 days.

9. Complete-case cohort construction

Complete cases are required for meanbp, hrt, resp, temp, wblc, crea, sod, death, d.time, dzgroup. No imputation is performed. The resulting cohort has 8,873 rows: 4,711 class 0 and 4,162 class 1; 232 rows are excluded.

19. Limitations

This is an observational cohort of seriously ill hospitalized adults from a specific historical clinical setting. Complete-case restriction can alter cohort composition. The endpoint collapses time-to-event information to a single horizon, and smoke-mode simulations are pipeline validation rather than stable inferential evidence. Disease group is retained for stratification but intentionally excluded from predictors.
